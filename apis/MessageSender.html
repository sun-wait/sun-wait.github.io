MessageSender API 文档

1. 概述

该API提供了通过CAN总线发送集群消息的功能，支持GB、CLEA和CLEA_557三种平台。

2. 主要类和枚举

ClusterModule 枚举

定义集群模块类型。

枚举值 描述
AdasSetting ADAS设置模块
Adas ADAS模块
AmbientLight 环境灯模块
Boot 启动模块
Chime 提示音模块
Climate 空调模块
Guage 仪表模块
Setting 设置模块
Telltale 指示灯模块
VehicleControl 车辆控制模块
VehicleInfo 车辆信息模块
Warning 警告模块

TupleValueType 枚举

定义元组值类型。

枚举值 描述
int 整型值
real 实数值

MessageSender 类

消息发送主类，提供集群消息发送功能。

3. 主要方法

getInstance() [类方法]

获取MessageSender单例实例。

返回: MessageSender - MessageSender实例

getTable(module)

获取指定模块的表格数据。

参数: module - 模块名称(ClusterModule枚举值)

返回: dict - 模块表格数据

checkCalibrations(module, description)

检查校准值是否符合预期。

参数:

  • module - 模块名称(ClusterModule枚举值)
  • description - 功能描述

返回: bool - 校准值是否符合预期

resetMessage(ids)

重置指定ID的消息。

参数: ids - 要重置的消息ID列表

filterMessageList(module, conditions)

根据条件过滤消息列表。

参数:

  • module - 模块名称(ClusterModule枚举值)
  • conditions - 过滤条件字典

返回: list - 过滤后的消息列表

sendMessages(modules, messageLists, options, autoSwitch=False)

发送集群消息。

参数:

  • modules - 模块列表
  • messageLists - 消息列表(嵌套列表，每个子列表对应一个模块)
  • options - 选项列表(每个元素对应一个模块)
  • autoSwitch - 是否自动切换状态(默认False)

getTupleSignalValue(tupleValue)

获取元组类型的随机信号值。

参数: tupleValue - 元组值(类型,最小值,最大值)

返回: int/float - 随机信号值

4. 使用示例

4.1 发送单个模块消息

sender = MessageSender.getInstance()
sender.sendMessages(
    modules=[ClusterModule.Telltale.value],
    messageLists=[["left_turn_signal", "right_turn_signal"]],
    options=["on"]
)

4.2 发送多个模块消息

sender = MessageSender.getInstance()
sender.sendMessages(
    modules=[ClusterModule.Telltale.value, ClusterModule.Warning.value],
    messageLists=[["left_turn_signal", "right_turn_signal"], ["low_fuel_warning"]],
    options=["on", "on"]
)

4.3 使用自动切换

sender = MessageSender.getInstance()
sender.sendMessages(
    modules=[ClusterModule.Telltale.value],
    messageLists=[["left_turn_signal"]],
    options=[],
    autoSwitch=True
)

4.4 过滤消息列表

sender = MessageSender.getInstance()
filtered_messages = sender.filterMessageList(
    module=ClusterModule.Telltale.value,
    conditions={"telltaleType": "cyclic"}
)

5. 注意事项

  • 使用前需要确保CAN总线连接正常
  • 不同平台(GB/CLEA/CLEA_557)的消息ID和信号可能不同
  • filterMessageList方法目前只支持"AND"操作符，如需"OR"操作符，请使用文档中提到的变通方法
  • autoSwitch参数启用时，options参数将被忽略
