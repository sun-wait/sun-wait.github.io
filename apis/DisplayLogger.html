DisplayLogger API 文档

该模块提供了收集显示相关日志的功能，包括IVI(车载信息娱乐系统)日志和QNX系统日志。

类文档

DisplayLogger

显示日志收集器的主类，提供收集IVI和QNX显示相关日志的功能。

getInstance()
获取DisplayLogger的单例实例。
返回:
DisplayLogger的实例
startIviLog(caseFolder, asyncMode=True)
启动IVI日志收集任务。
caseFolder 日志保存的文件夹路径
asyncMode 是否异步执行，默认为True
返回:
布尔值，表示是否成功启动
startQnxLog(caseFolder, asyncMode=True)
启动QNX日志收集任务。
caseFolder 日志保存的文件夹路径
asyncMode 是否异步执行，默认为True
返回:
布尔值，表示是否成功启动
startLog(caseFolder, asyncMode=True)
同时启动IVI和QNX日志收集任务。
caseFolder 日志保存的文件夹路径
asyncMode 是否异步执行，默认为True
waitQnxLogFinish()
等待QNX日志任务完成(仅异步模式下使用)。
waitIviLogFinish()
等待IVI日志任务完成(仅异步模式下使用)。
waitLogFinish()
等待所有日志任务完成(仅异步模式下使用)。
IviLogTask

IVI日志收集任务线程类，内部使用。

run()
执行IVI日志收集任务，包括收集dumpsys window和dumpsys SurfaceFlinger信息。
QnxLogTask

QNX日志收集任务线程类，内部使用。

addPermissionToScripts()
为远程脚本添加执行权限。
返回:
布尔值，表示是否成功
run()
执行QNX日志收集任务，包括上传脚本、添加权限、执行脚本、拉取日志和清理远程文件。

常量说明

  • REMOTE_DISPLAY_SCRIPTS_PATH: 远程脚本存放路径，默认为"/ota_update/display"
  • EXECUTE_DISPLAY_SCRIPT_CMD_TIMEOUT: 执行显示脚本的超时时间，默认为300秒
  • PULL_OUT_DISPLAY_QNX_LOG_TIMEOUT: 拉取QNX日志的超时时间，默认为60秒

使用示例


# 同步模式收集日志
logger = DisplayLogger.getInstance()
logger.startLog("/path/to/logs", asyncMode=False)

# 异步模式收集日志
logger.startLog("/path/to/logs")
# 其他操作...
logger.waitLogFinish()
