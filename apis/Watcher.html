Watcher API 文档

该模块提供了设备界面元素监控功能，可以监听特定元素的变化并触发回调。

主要功能：
  • 添加界面元素监听器
  • 启动/停止监控
  • 重置监控状态
  • 检查监控状态

类文档

Watcher

界面元素监控类，负责管理设备界面元素的监控任务。

__init__(self)
构造函数，初始化监控器。
getInstance(cls)
获取Watcher的单例实例。

返回: Watcher实例

addWatcher(self, device, indicator, callback)
添加界面元素监控器。

参数:

  • device: 设备对象
  • indicator: 监控指标(文本或xpath字符串)
  • callback: 触发回调函数
start(self, device)
启动所有监控器。

参数:

  • device: 设备对象
reset(self, device)
重置所有监控器。

参数:

  • device: 设备对象
stop(self, device)
停止所有监控器。

参数:

  • device: 设备对象
isWorking(self)
检查监控器是否正在运行。

返回: 布尔值，表示监控器是否正在运行

使用示例

# 获取Watcher实例
watcher = Watcher.getInstance()

# 定义回调函数
def my_callback():
    print("元素被触发!")

# 添加监控器
watcher.addWatcher(device, "确定按钮", my_callback)

# 启动监控
watcher.start(device)

# 检查监控状态
if watcher.isWorking():
    print("监控器正在运行")

# 停止监控
watcher.stop(device)

注意事项

  • 需要有效的设备连接
  • indicator可以是文本内容或xpath表达式
  • 回调函数会在监控元素出现时触发
  • 使用单例模式确保全局只有一个监控器实例
