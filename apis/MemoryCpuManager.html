MemoryCpuManager API 文档

该文档描述了 MemoryCpuManager 类的API接口，用于管理Android和QNX系统的内存和CPU信息收集。

类概述

MemoryCpuManager 是一个单例类，提供了以下功能：

  • 准备收集环境（上传脚本文件等）
  • 启动Android和QNX系统的信息收集进程
  • 检查收集进程是否正在运行
  • 停止收集进程
  • 将收集的数据从设备拉取到本地

类方法

getInstance()

获取 MemoryCpuManager 的单例实例。

返回: MemoryCpuManager 实例
prepare(senario)

准备收集环境，包括上传必要的脚本文件到Android和QNX设备。

参数:
  • senario - 场景类型，决定使用哪种收集脚本
返回: 操作结果（可能为None）
dumpAndroidInfos(breakPointResume)

启动Android系统的信息收集进程。

参数:
  • breakPointResume - 是否从断点恢复收集
dumpQnxInfos(breakPointResume)

启动QNX系统的信息收集进程。

参数:
  • breakPointResume - 是否从断点恢复收集
startDump(casePath, breakPointResume=False)

同时启动Android和QNX系统的信息收集。

参数:
  • casePath - 保存收集数据的本地路径
  • breakPointResume - 是否从断点恢复收集（默认为False）
返回: 布尔值，表示操作是否成功
checkAndroidShellProcess()

检查Android系统的收集进程是否正在运行。

返回:
  • 1 - 进程正在运行
  • 0 - 进程未运行
  • -1 - 检查失败
checkQnxShellProcess()

检查QNX系统的收集进程是否正在运行。

返回:
  • 1 - 进程正在运行
  • 0 - 进程未运行
  • -1 - 检查失败
stopShellProcess()

停止Android和QNX系统的收集进程。

stopDump(casePath=None, stopShell=True)

停止收集并将数据从设备拉取到本地。

参数:
  • casePath - 保存收集数据的本地路径（可选）
  • stopShell - 是否停止收集进程（默认为True）
返回: 布尔值，表示操作是否成功

常量

  • PUSH_PREPARE_FILE_TIMEOUT - 上传准备文件超时时间（60秒）
  • PULL_OUT_DATAS_TIMEOUT - 拉取数据超时时间（2000秒）
  • QNX_REMOTE_DUMP_LOGS_FILE - QNX远程日志转储脚本路径
  • QNX_REMOTE_DUMP_SCRIPT_FILE - QNX信息收集脚本路径
  • QNX_LOG_PATH - QNX日志存储路径

注意事项

  • 使用前需要确保设备已正确连接（ADB和SSH）
  • 需要预先准备好收集脚本文件
  • 收集的数据会保存在指定的本地路径中
  • 该类实现了单例模式，应通过getInstance()方法获取实例
